# 账号管理器（Account Manager）需求文档

## 1. 文档信息

- 文档状态：需求草案
- 产品形态：浏览器插件、同步服务端、服务端管理后台
- 开源范围：插件端、服务端和管理后台均开源
- 核心原则：离线优先、客户端加密、服务端仅可见安全默认元数据

## 2. 产品目标

开发一套可自托管的账号管理系统（Account Manager）。用户可以在浏览器插件中离线管理并生成 TOTP 验证码，也可以连接远程服务，将加密后的数据同步到服务端，并在另一台设备上使用相同的远程账号和同步密码恢复数据。

系统必须保证：

1. 没有网络时，插件仍可查看条目并生成验证码。
2. TOTP 密钥等敏感数据只在客户端加密和解密。
3. 服务端不接收同步密码，也不具备解密敏感数据的能力。
4. 远程账号密码与同步密码相互独立，承担不同职责。
5. 服务端可以保存同步所需的非敏感元数据和经过用户确认的账号/关系运营元数据，
   但不能看到可接管账号的秘密正文。

## 3. 核心概念

### 3.1 远程账号密码

远程账号密码用于登录同步服务，证明用户身份并获得同步权限。

- 账号密码提交给服务端进行身份认证。
- 服务端只能保存安全的密码哈希，不得保存明文密码。
- 修改或重置账号密码不应改变已同步数据的加密方式。
- 管理员可以禁用账号的同步权限，但不能因此解密用户数据。

### 3.2 同步密码

同步密码用于在插件端加密和解密需要同步的敏感数据。

- 同步密码由用户在插件端输入。
- 同步密码不得发送到服务端，不得出现在请求 URL、请求头、日志或遥测信息中。
- 同步密码需要保存在本地，以支持后续自动加密、解密和同步。
- 同步密码不得以明文形式写入本地存储。
- 插件应保存由同步密码保护的本地密钥材料，或保存经过本地安全包装的同步凭证。
- 插件解锁后，解密所需密钥可以短期保存在内存中；锁定、退出浏览器或达到自动锁定时间后应清除。
- 用户可以选择退出远程账号时是否同时清除本地保存的同步凭证。
- 如果用户丢失同步密码且没有可用设备或恢复材料，服务端无法帮助恢复敏感数据。

具体密钥派生算法、加密算法和本地安全存储方案在技术设计阶段确定，本需求不规定实现常量。

### 3.3 账号秘密与服务端可见元数据

以下信息属于敏感数据，必须在客户端加密后才能上传：

- TOTP secret；
- 密码、恢复码、Cookie、token、API key；
- 完整邮箱、完整手机号、代理认证信息；
- 用户备注；
- 恢复信息；
- 其他能够登录、恢复或接管账号的字段。

服务端可以保存账号类型、平台、状态、标签、关系类型、掩码后的登录标识、是否配置密码/2FA/手机号/代理等布尔元数据。
安全默认实现不得上传完整登录标识或完整手机号/代理的普通 SHA-256 等可枚举哈希。

### 3.4 分组

用户可以创建分组，对 2FA 条目进行分类和切换查看。

服务端可以保存分组信息，但需要区分两类数据：

- 可明文保存：随机分组 ID、排序序号、版本号、更新时间、删除标记等同步元数据。
- 默认加密保存：分组名称、描述、图标、颜色和其他用户自定义内容。

原因是“工作账号”“生产环境”等分组名称可能泄露用户身份和使用场景。首个版本默认将分组展示信息放入加密数据中；如果未来允许明文分组名称，必须由用户明确开启并说明隐私影响。

## 4. 用户角色

### 4.1 普通用户

- 在插件中管理本地 2FA 数据；
- 注册、登录和退出远程账号；
- 设置、更新和清除本地同步密码；
- 手动或自动执行上传、下载和同步；
- 管理分组并切换当前分组；
- 查看已连接设备并撤销设备同步权限。

### 4.2 服务管理员

- 登录管理后台；
- 查看和管理用户账号状态；
- 查看设备数量、最近同步时间、存储用量和服务运行状态；
- 禁用用户或撤销异常会话；
- 查看不包含敏感内容的审计日志。

管理员不能查看同步密码、TOTP secret、密码、完整登录标识、完整手机号、代理认证、分组名称或其他已加密内容。

## 5. 插件端功能需求

### 5.1 本地使用

- 支持手动添加、编辑和删除 TOTP 条目。
- 支持识别标准 `otpauth://` 数据。
- 支持通过二维码导入 TOTP 条目。
- 显示当前验证码、剩余有效时间，并支持复制。
- 插件断网时仍可完成 TOTP 生成和本地数据管理。
- 插件应检测明显的本机时间异常，并提示 TOTP 可能不准确。

### 5.2 本地安全

- TOTP 敏感数据不得明文保存到浏览器存储。
- 插件必须具有锁定和解锁状态。
- 插件锁定时不得展示敏感条目或生成可复制的验证码。
- 支持配置自动锁定时间。
- 不使用浏览器自带的跨设备扩展存储同步敏感数据。
- 插件权限应遵循最小权限原则。
- 当前阶段账号卡片以点击复制为主；若后续开放网页填充入口，只允许由用户在扩展侧边栏手动触发，不得静默自动填充或后台填充。

### 5.3 分组与切换

- 支持创建、重命名、排序和删除分组。
- 支持将条目移动到其他分组。
- 支持切换当前分组以及查看全部条目。
- 删除非空分组前必须让用户选择移动或删除组内条目。
- 分组及条目的变更都应参与同步和冲突处理。

### 5.4 远程连接

- 用户输入服务端地址、账号和账号密码进行登录。
- 登录成功后，插件保存必要的服务地址、用户标识和会话信息。
- 会话失效不影响本地离线使用，只暂停远程同步。
- 服务端证书异常或连接不安全时，插件不得上传敏感数据。

### 5.5 同步设置

- 用户连接远程账号后，需要输入同步密码才能上传或下载敏感数据。
- 首次使用远程空仓库时，插件加密本地数据并上传。
- 首次连接已有远程仓库时，插件下载密文并使用同步密码解密。
- 同步密码错误时，插件必须明确提示解密失败，不得覆盖本地或远程数据。
- 支持手动同步；自动同步作为可配置功能。
- 显示上次同步时间、当前状态和失败原因。

### 5.6 多配置切换

首个版本中的“切换”定义为分组切换。多个完全独立的密钥库、多个远程账号同时在线或团队共享不属于首个版本范围。

### 5.7 UI/UX 与交互要求

- 界面定位：插件专为高频、快速、单手或键盘单键操作优化。核心操作（查看验证码、一键复制、切换分组、关键字过滤）应在 1-2 次点击或按键内完成。
- 尺寸约束：插件当前以 Chromium Side Panel 形态运行，最小宽度为 320px，高度填满侧边栏视口。长列表时容器内部滚动，严禁触发浏览器宿主窗口或侧边栏本身的滚动条。
- 信息层级：首屏上方展示当前分组、时间倒计时进度条、搜索框与新增按钮；主体区域展示 TOTP 条目列表（第一优先展示发行方、用户名、当前验证码和复制按钮）；非核心设置、账号同步等入口收纳于底部栏或侧滑抽屉。
- 交互反馈：
  - 复制反馈：点击 TOTP 验证码或复制按钮后，必须在 200ms 内给出即时视觉反馈（如按钮变为已复制状态、显示淡入淡出的微型 Toast “已复制”），并在 1.5s 后自动复原，不阻塞用户的后续高频操作。
  - 时间警示：当 TOTP 剩余有效时间少于 5s 时，进度条或倒计时数字转换为警示色（如橙红色），动态提醒验证码失效临界点。
  - 动画曲线：状态切换、分组折叠、侧滑菜单展开等过渡动效持续时间严格控制在 150ms - 300ms，使用流畅的缓动曲线（如 ease-in-out），严禁出现瞬间闪现或卡顿抖动。
- 状态设计：
  - 加载中状态：数据加载时使用与实际列表卡片排版高度、位置严格一致的骨架屏（Skeleton Screen）占位，避免加载完成后页面排版发生抖动。
  - 空状态：当分组内无数据或本地无条目时，展示包含指引性文案（如“暂无条目，点击下方按钮添加首个验证码”）和高亮操作按钮的图形化空状态。
  - 错误状态：同步或网络异常时，在底部/顶部以非阻塞的轻量警告条（Alert Banner）呈现具体错误（如“同步密码错误”），并提供即时重试按钮，不打断本地已解锁的 TOTP 正常显示。

## 6. 同步需求

### 6.1 基本流程

1. 用户使用远程账号密码登录服务端。
2. 插件读取本地已保存并受保护的同步凭证，必要时要求用户解锁。
3. 插件在本地序列化并加密敏感数据。
4. 插件将密文、随机分组 ID 和同步版本等信息上传服务端。
5. 另一台设备登录同一远程账号并输入相同同步密码。
6. 新设备下载密文，在本地解密后写入本地数据库。

### 6.2 同步数据边界

服务端可以读取：

- 用户 ID；
- 设备 ID；
- 随机密钥库 ID 和随机分组 ID；
- 数据版本、更新时间和删除标记；
- 密文大小、同步状态和必要的存储元数据。

服务端不能读取：

- 同步密码和解密密钥；
- TOTP secret；
- 完整登录标识、密码、TOTP secret、完整手机号、代理认证；
- 默认加密的分组名称、描述等展示信息；
- 用户备注和恢复信息。

### 6.3 冲突处理

- 服务端应通过版本号或修订号拒绝直接覆盖过期版本。
- 插件发现本地和远程均有更新时，应先保留两边数据，再进行合并或提示用户选择。
- 不得在检测到冲突时静默覆盖整个密钥库。
- 删除操作需要同步删除标记，避免旧设备将已删除条目重新上传。

详细的条目级合并规则在同步协议设计文档中定义。

### 6.4 同步密码变更

- 用户必须先使用旧同步密码解密本地数据。
- 插件使用新同步密码重新包裹同一个密钥库 DEK，不重新加密每条记录正文。
- 新信封上传成功后，其他设备下次同步时需要输入新同步密码解包同一个 DEK。
- 其他设备下次同步时需要输入新同步密码。
- 服务端不提供同步密码重置或明文恢复功能。

## 7. 服务端功能需求

- 提供用户注册、登录、退出和账号密码修改接口。
- 提供设备注册、设备撤销和会话管理接口。
- 提供密文上传、下载、版本检查和删除接口。
- 严格隔离不同用户的数据访问权限。
- 对认证和同步接口实施速率限制。
- 所有外部接口必须通过安全传输协议提供。
- 日志不得记录账号密码、同步密码、解密密钥或上传密文正文。
- 支持数据库备份和恢复，但备份中的用户敏感数据仍应为客户端密文。
- 提供适合自托管的部署和升级方式。

## 8. 管理后台需求

- 管理员独立登录。
- 查看用户列表、账号状态、注册时间和最近同步时间。
- 查看用户设备数量、存储用量和同步请求统计。
- 启用或禁用用户同步权限。
- 撤销指定设备或会话。
- 查看管理员操作日志和异常认证日志。
- 查看服务版本、数据库状态和基础运行指标。
- 不提供查看、搜索、导出或修改用户明文 2FA 数据的能力。
- 可以查看服务端可见的账号/关系元数据和只读关系图，但不得返回或展示账号秘密密文正文。
- UI/UX 与响应式布局：
  - 界面定位：专为高密度的系统配置与运维监控设计，优化网格排版，确保单屏内展示尽可能多的审计和状态信息。
  - 响应式布局：管理后台自适应 320px 至 2560px 宽度的显示设备。
    - 桌面端（宽屏，≥ 1024px）：采用左侧常驻导航栏、多列数据表格、右侧折叠详情抽屉（Drawer）的三栏经典布局。
    - 平板端（中屏，768px - 1023px）：左侧导航自动收起为悬浮折叠菜单，数据表格支持横向滚动，详情展示变更为模态弹窗。
    - 移动端（窄屏，< 768px）：导航栏收起为顶部通栏汉堡菜单；复杂多列表格自动转为单列卡片流（Card List）布局或横向手势滑动；敏感操作（如禁用、撤销）收纳至行末的“更多”菜单中，防止误触。确保在所有小屏设备下内容无溢出截断和组件重叠。
  - 交互反馈与操作安全：对禁用用户、撤销设备、强制会话下线等敏感操作，必须弹出带有警示标志的二次确认模态对话框，且“确认”操作使用高对比度的红色按钮；所有表单与表格筛选必须提供局部加载动画，且操作结果通过屏幕右上角自动淡出的全局通知（Toast）提示，持续时间 3s。
  - 状态设计：列表加载中展示占位 Spinner，无数据时展示标准的无数据占位图，API 报错时显示带重试引导的错误卡片。

## 9. 关键用户流程

### 9.1 首台设备

1. 安装插件并创建本地密钥库。
2. 添加和分组 TOTP 条目。
3. 输入远程地址、账号和账号密码完成连接。
4. 设置同步密码并确认本地保存方式。
5. 插件加密数据并执行首次上传。

### 9.2 第二台设备恢复

1. 安装插件。
2. 输入相同的远程地址、账号和账号密码。
3. 下载远程密文。
4. 输入相同的同步密码。
5. 插件在本地完成解密，展示分组和 TOTP 条目。

### 9.3 离线使用

1. 设备无法连接远程服务。
2. 用户解锁本地插件。
3. 插件使用本地数据生成验证码。
4. 本地变更进入待同步状态，联网后再同步。

## 10. 非功能需求

### 10.1 安全

- 客户端加密和服务端认证必须使用经过广泛审计的标准方案。
- 不自行设计新的加密算法。
- 对插件、服务端 API 和管理后台进行独立威胁建模。
- 敏感操作和错误信息不得泄露密钥材料。
- 正式发布前应进行安全审计。

### 10.2 兼容性

- 优先支持基于 Chromium 的浏览器。
- Firefox 支持范围在技术验证后确定。
- 同步数据格式必须包含版本信息，以支持客户端升级。

### 10.3 可维护性与开源

- 插件、服务端和管理后台分别提供构建、测试和部署说明。
- 服务端支持自托管。
- 仓库不得包含生产凭证或默认弱密码。
- 开源许可证在开发启动前确定。

### 10.4 可用性

- 非专业用户可以区分账号密码和同步密码。
- 设置同步密码时必须解释其用途和丢失后的后果。
- 网络或同步失败不得阻塞本地生成验证码。

### 10.5 界面一致性与无障碍访问

- 统一设计系统（Design System）：
  - 必须建立和使用统一的视觉设计系统，定义明确的主题色（Primary）、辅助色（Secondary）、中性色（Neutral）和语义色（Success, Warning, Danger, Info）调色板。
  - 间距、内边距（Padding）和外边距（Margin）应优先遵循以 4px 或 8px 为基准的网格系统（如 4, 8, 12, 16, 24, 32px）；确有特殊布局需要时，应通过统一设计令牌定义并说明用途。
  - 所有组件的字体字号、圆角大小和阴影高度必须来自全局 Design Tokens，确保插件与管理后台拥有统一的视觉风格。
- 无障碍访问（Accessibility / A11y）：
  - 键盘导航：所有可交互元素（按钮、输入框、下拉菜单等）必须支持完整的键盘 Tab 键顺序聚焦导航，聚焦时必须显示高对比度的可见焦点环（Focus Ring，对比度不低于 3:1，宽度不低于 2px）；必须支持使用 Enter 或 Space 键触发交互，使用 Esc 键关闭所有打开的弹窗、侧滑菜单和模态框。
  - 屏幕阅读器支持：所有纯图标/无文本的交互元素（如复制、删除、设置等按钮）必须配置语义明确的 `aria-label` 属性；所有表单输入项必须有对应的 `<label>` 元素或通过 `aria-labelledby` 进行关联。
  - 色彩对比度：界面中所有正文文本、关键指示器与背景色的色彩对比度必须符合 WCAG 2.1 AA 级标准（普通文本对比度至少为 4.5:1，大文本和关键图形元素对比度至少为 3:1）。

## 11. 首个版本范围

### 11.1 包含

- 本地 TOTP 管理和离线生成；
- 二维码和 `otpauth://` 导入；
- 分组、排序和切换；
- 本地加密存储和插件锁定；
- 远程账号认证；
- 使用独立同步密码进行客户端加密同步；
- 第二台设备下载和恢复；
- 设备撤销；
- 基础用户、设备、审计和运行状态管理后台；
- 插件、服务端、管理后台开源及自托管说明。

### 11.2 暂不包含

- 团队或家庭共享；
- 多用户共同编辑同一密钥库；
- 企业 SSO 和复杂 RBAC；
- 移动端应用；
- 浏览器网页自动填充；当前阶段保留 content script 基础能力，但侧边栏 UI 暂未开放填充入口；
- 服务端恢复同步密码；
- 多个独立密钥库或多个远程账号同时在线；
- 复杂历史版本与回滚功能。

## 12. 验收标准

1. 断开网络后，已解锁插件仍能生成正确的 TOTP 验证码。
2. 用户可以创建分组、移动条目并切换分组查看。
3. 用户可以使用账号密码登录远程服务，并使用独立同步密码上传密文。
4. 网络抓包和服务端日志中不出现同步密码、解密密钥或 TOTP secret。
5. 数据库管理员只能看到密文和允许公开的同步元数据，不能得到条目及分组的默认明文展示信息。
6. 第二台设备使用相同账号密码和同步密码后，可以恢复分组及 TOTP 条目。
7. 输入错误同步密码时，客户端不覆盖本地和远程数据。
8. 服务端不可用或账号被禁用时，本地离线功能仍可使用。
9. 管理员可以禁用同步账号和撤销设备，但不能查看用户明文数据。
10. 两台设备产生并发更新时，系统能够检测冲突，不会静默丢弃其中一方的数据。
11. 插件以 Chromium Side Panel 展示时，最小宽度不低于 320px，高度填满侧边栏视口，且长列表条目在容器内正常滚动，无宿主窗口滚动条。
12. 插件一键复制功能在触发后 200ms 内提供“已复制”状态的视觉反馈，并在 1.5s 内自动复原。
13. 管理后台在 375px（移动端）和 1920px（桌面端）视口宽度下进行兼容性测试，页面元素无溢出截断、重叠和无法点击的现象，移动端下可通过汉堡菜单和卡片流完成全部管理功能。
14. 所有交互元素支持完整的键盘 Tab 导航，聚焦环（Focus Ring）的视觉对比度不低于 3:1，且按 Esc 键能成功关闭所有打开的模态对话框与下拉菜单。
15. 界面中所有文本与背景色对比度在 Chrome DevTools 的 Accessibility 检查中通过 WCAG 2.1 AA 级标准审计。
16. 所有页面加载和列表刷新操作均配置了防抖动的骨架屏或局部加载状态，数据加载完成前后的累计布局偏移（CLS）指标小于 0.1。
17. 颜色、间距、字体、圆角和阴影等核心视觉参数通过统一设计令牌管理；代码检查确认不存在未经定义或说明的重复硬编码视觉参数。

## 13. 待确认事项

- 本地同步密码的默认保存策略：每次浏览器启动重新输入，还是允许用户选择持久保存。
- 是否需要账号密码找回功能，以及找回方式。
- 是否保留有限数量的密文历史版本以防止误覆盖。
- Chromium 与 Firefox 是否在首个版本同时发布。
- 采用单仓库还是插件端、服务端和管理后台分仓库。
- 最终开源许可证。
